# 更新日志

## [2.0.0] - 2026-03-31

### 新增功能 ✨
- **自动刷新**: 仪表盘每30秒自动刷新数据
- **分页支持**: API 支持分页查询 (`?page=1&limit=50`)
- **加载状态**: 显示加载动画和最后更新时间
- **结构化日志**: 使用 JSON 格式的结构化日志系统
- **配置验证**: 启动前自动验证环境变量和配置

### 算法改进 🧠
- **关键词匹配优化**:
  - 添加位置权重（早出现的关键词权重更高）
  - 添加频率权重（多次出现的关键词加分）
  - 改进评分算法，更准确的分类
  
- **情感分析增强**:
  - 支持否定词检测（"不好" → 负面）
  - 支持表情符号识别（😊😡等）
  - 支持标点符号分析（感叹号增强情感）
  - 新增更多情感关键词
  
- **紧急度检测改进**:
  - 新增更多紧急关键词
  - 支持多个模式累积评分
  - 支持连续感叹号检测

- **关键词提取优化**:
  - 扩充停用词列表
  - 按频率排序返回
  - 过滤纯数字

### 性能优化 ⚡
- **数据缓存**: 5秒TTL缓存，减少文件系统访问
- **数据限制**: 
  - 最多读取最近7天的分类文件
  - 单次查询最多返回1000条消息
- **HTTP请求**: 使用 undici 替代 curl 命令
- **错误重试**: 指数退避策略，避免频繁失败

### 可靠性提升 🔒
- **全面错误处理**:
  - 所有关键路径添加 try-catch
  - 文件操作错误不会导致崩溃
  - 网络错误自动重试
  
- **错误恢复**:
  - 单条消息处理失败不影响其他消息
  - 连续失败10次后优雅退出
  - 保存数据失败记录日志但继续运行

- **日志系统**:
  - 支持 debug/info/warn/error 级别
  - JSON 格式便于解析
  - 包含时间戳、错误堆栈等完整信息

### 技术改进 🛠️
- **依赖更新**: 使用 undici 7.x 替代 child_process.spawn('curl')
- **类型安全**: 添加完整的 TypeScript 类型定义
- **代码组织**: 
  - 新增 logger.ts 日志模块
  - 新增 config-validator.ts 配置验证模块
  - 改进代码注释和文档

### 破坏性变更 ⚠️
- `/api/messages` 现在返回分页格式:
  ```json
  {
    "data": [...],
    "pagination": {
      "page": 1,
      "limit": 50,
      "total": 100,
      "totalPages": 2,
      "hasNext": true,
      "hasPrev": false
    }
  }
  ```
- 日志格式从纯文本改为 JSON 格式

### 修复 🐛
- 修复大数据量时内存溢出问题
- 修复代理配置不生效的问题
- 修复 curl 命令在某些系统上不可用的问题
- 修复情感分析对否定词判断错误的问题
